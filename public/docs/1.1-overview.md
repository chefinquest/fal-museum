# FAL Museum

FAL Museum is a browser-first Three.js gallery with a square floor plan, a central dividing wall, six exhibition surfaces, and one generated artwork per surface. The user walks the whole museum in first person with smooth, natural pointer-lock navigation.

## Overview

The museum is designed as a realistic compact gallery: white brick walls, warm wood flooring, a textured ceiling, baseboards, picture lights, framed works, labels, and a few sculptural details for scale. The floor plan is a square with one middle partition wall, creating six curated wall surfaces: north, east, south, west, partition-left, and partition-right.

The experience is intentionally lightweight: the scene uses procedural materials, primitive geometry, static generated images, and direct collision math instead of a heavy physics engine. This keeps first-person navigation fast and responsive while preserving the visual cues that make the gallery feel believable.

## Features

- **Walkable first-person museum:** Click to enter pointer lock, use WASD to walk, mouse to look, Shift to sprint, and Escape to unlock.
- **Square gallery layout:** Four exterior walls plus two sides of the central partition create six wall surfaces total.
- **Six FAL artworks:** Every wall has a framed artwork generated through a different FAL endpoint and style direction.
- **Artwork labels:** Each frame includes a short plaque with the artwork title and model name.
- **Realistic rendering:** ACES tone mapping, textured materials, fog, shadows, spotlights, baseboards, picture frames, sculptures, and warm museum lighting.
- **Fast browser performance:** Procedural textures, optimized geometry, bounded DPR, preloaded images, and lightweight mesh details keep the scene responsive.
- **Docs route:** This documentation is available from `/docs` in the same app.

## User Guide

### Getting Started

Open `/`, click **Click to walk the museum**, and use the mouse and keyboard as if walking through a small gallery. The visitor starts near one side of the room and can loop around either end of the central partition to inspect all six framed works.

### Key Actions

| Action | Control | Notes |
|--------|---------|-------|
| Enter museum | Click the center overlay | Enables browser pointer lock |
| Move | WASD or arrow keys | Smooth accelerated movement |
| Look | Mouse | Active only while pointer lock is enabled |
| Sprint | Shift | Raises speed but preserves collision |
| Exit pointer lock | Escape | Restores the cursor |
| Open docs | `/docs` or HUD link | Leaves WebGL scene route |

### Tips & Shortcuts

- Walk around both ends of the middle partition to view the two partition-facing artworks.
- Stand several steps away from each frame to see the plaque and image together.
- Press Escape if the cursor needs to be released for normal browser interaction.

## Technical Details

### Application Stack

| Layer | Technology | Purpose |
|-------|------------|---------|
| App shell | React + TypeScript | Route selection, UI overlays, docs renderer |
| 3D renderer | Three.js + React Three Fiber | Declarative WebGL scene graph |
| Helpers | Drei | `Html`, `Preload`, texture loading, soft shadows |
| Content | FAL generated static assets | Six unique artwork images |
| Docs | Markdown + marked | ryOS-style documentation route |
| Build | Vite | Fast development and static production build |

### Primary Files

| File | Responsibility |
|------|----------------|
| `src/App.tsx` | Museum scene, player controls, artwork placement, and docs renderer |
| `src/App.css` | Gallery HUD, labels, and docs styling |
| `src/artworks.json` | FAL artwork metadata and image paths |
| `public/artworks/` | Downloaded generated artwork assets |
| `public/docs/` | Markdown documentation rendered at `/docs` |
